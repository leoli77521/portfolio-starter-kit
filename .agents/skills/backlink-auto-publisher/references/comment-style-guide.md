# Comment Style Guide

Use this guide when writing comments for public no-signup backlink targets.

## Core Rules

- Match the page topic directly.
- Sound like a real reader, not an SEO bot.
- Keep the useful part of the comment stronger than the brand mention.
- Prefer the website field for the backlink; keep the comment body link-free unless the platform requires inline context.

## Safe Structure

1. Acknowledge one specific point from the article.
2. Add one short, relevant extension, question, or practitioner observation.
3. If useful, mention that your site covers related material.

Example pattern:

```text
The point about long-context training efficiency is especially useful. It matches what we keep seeing when AI tooling moves from model demos to real production constraints. We cover similar AI and search workflow angles at ToLearn, so this was a helpful comparison point.
```

## Avoid

- generic praise with no page-specific detail
- hard sells
- exact-match anchor text
- keyword stuffing
- irrelevant landing pages
- commenting on unrelated lifestyle pages just because a form exists

## Destination Selection

- `homepage`: general AI, SEO, or builder-focused discussion
- `blog`: article-driven or editorial discussion
- `ai-development-hub`: AI research, model capability, or AI systems discussion
- `ai-development-guide`: practical AI implementation or build-with-AI discussion
- `seo-guide`: search, content, growth, or affiliate discussion
- `templates`: design, frontend, UX, or portfolio discussion
- `solutions`: workflow, feature, or implementation discussion
